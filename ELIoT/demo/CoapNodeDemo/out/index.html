Rot.js with Node-Coap

This is an simulation example based on ELIOT Platform. Through implementing LWM2M communication, robot node can register and update it location through respective SmartObject.

Prerequisite

  • Docker (Version 17 or newer).
  • Leshan 0.1.11 M5 SNAPSHOT or 0.1.11 M4
  • npm
  • JsDoc (For developers)

Usage

Client:

./run.sh

Server:

docker build -t server .
docker run --rm -ti --name mi server

Client Developing Guide

Client is devepled based on Coap-node project from npm in Node.js. The communication flow is designed according to [RFC CoAP][https://tools.ietf.org/html/rfc725].

Some principles to follow when expanding the client / create a new one:

Creat new SmartObject

IPSO defines a hierarchical data model to describe real-world gadgets, such as temperature sensors and light controllers, which is called SmartObject . In our use case, each smart object will be simulated node with unique IP address and registration ID.

Client Initiative Request

For various use case, user may need to let client initiate a request. In such case, we will adopt Coap-node library.
